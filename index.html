Functional Programming in Swift

Functional Programming has a lot of key benefits that can improve your development process by avoiding side effects and providing referential transparency. Your code will be cleaner, safer, and easier to test. We decided to extend the Swiftz framework to add a lot of the syntax and features of the Scala language to Swift. We’ve been enjoying the benefits of Scala in our projects here in 47 Degrees for some years now, and we’d love Second Bridge to be your gateway to Functional Programming in your iOS apps!

New Features

Second Bridge comes packaged with a set of selected features from the Scala language:

  • Immutable data-types that include Lists, Maps, Vectors, Stacks… and protocols derived from the Traversable and Iterable traits from Scala that will help you to build your own data-types in no time.
  • Partial functions and algorithm operators based on them.
  • Pattern matching even more powerful than Swift’s.

Playing with Playgrounds

We know that Functional Programming can be a little confusing for some developers used to the more tradicional imperative paradigm. So we wanted to help other iOS developers to learn how to use all these features.

Playgrounds are a great tool for users to play with the new types and functions. Second Bridge ships with several Playgrounds showcasing several key features, so you can just start writing functional code and seeing results right away.
